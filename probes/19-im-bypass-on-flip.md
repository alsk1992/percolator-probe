# Probe 19 — IM bypass on position flip

**Date**: 2026-04-23
**Category**: Margin (#19 in author's 49-category checklist)
**Verdict**: DISCARDED

## Hypothesis

Classic perp-DEX exploit. When a trader flips from long to short (or
vice versa) in a single trade, the post-trade position is a fresh
opening of the opposite direction. Initial Margin (IM > MM typically)
should fire, not Maintenance Margin. If the engine incorrectly treats
a flip as "just a size change" and runs only MM, a trader can open an
under-IM position by routing through zero.

## Code path

`percolator/src/percolator.rs:4220 enforce_one_side_margin`.

The risk-increasing classifier at 4261–4266:

```rust
let risk_increasing = abs_new > abs_old
    || (*old_eff > 0 && *new_eff < 0)   // long → short: IM
    || (*old_eff < 0 && *new_eff > 0)   // short → long: IM
    || *old_eff == 0;                    // flat → open: IM
```

Both sign-flip cases are explicitly enumerated as risk-increasing.
Line 4276 then invokes `is_above_initial_margin_trade_open`, which at
line 3035 uses `Eq_trade_open_raw` — the counterfactual equity with
the candidate trade's positive slippage removed (spec §3.5). So a
trader cannot use positive slippage generated by the flip itself to
satisfy IM.

Additional floor: line 3042 takes `max(proportional, min_nonzero_im_req)`,
so even tiny positions hit a minimum IM requirement.

## Why discarded

Flip is correctly classified as risk-increasing. IM fires with
positive-slippage-excluded equity. The `abs_new > abs_old` OR sign-flip
OR old==0 disjunction covers every path where a trader could attempt
to gain or pivot exposure. No flip path reaches the MM or
`strictly_reducing` branches.

## What would a bug look like

Any of these would be exploitable:

1. Flip classified as `strictly_reducing` (not the case — 4268 requires
   same sign on both sides)
2. Flip using pre-trade equity (ignores loss from the flip trade itself
   — not the case, `candidate_trade_pnl` is passed in)
3. Flip allowing positive candidate slippage in IM numerator (not the
   case — `Eq_trade_open_raw` zeroes out positive PnL per spec §3.5)
4. MM-passes-therefore-trade-allowed bypassing IM for flips (not the
   case — 4280 MM fast-path only reachable when `risk_increasing == false`)

None of these hold.
